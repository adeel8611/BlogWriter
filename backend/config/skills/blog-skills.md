# Blog Writing Skills - QUALITY STANDARDS (8+ Out of 10)

## Skill 1: Structure & Organization (CRITICAL)

### Description
Create perfectly structured blogs that readers can scan and understand in seconds.

### REQUIRED ELEMENTS
1. **H1 Title** (# Title)
   - Benefit-driven, keyword-rich
   - 8-12 words optimal
   - Clear value proposition
   - Example: "10 Best Budget-Friendly Gifts for Padel Players"

2. **Meta Description** (160 characters)
   - First line after title
   - Benefit statement with keyword
   - Example: "Discover 10 budget-friendly gifts for padel players, including useful accessories and smart ideas perfect for beginners and regular players."

3. **Introduction** (150-200 words)
   - Hook: Opens with relatable insight (e.g., "With more people picking up a padel racket...")
   - Context: Explains why this topic matters
   - Promise: Tells reader what they'll learn
   - DO NOT be generic ("In today's world...")

4. **Main Content** (5-8 H2 sections, each 150-200 words minimum)
   - Clear H2 headers
   - Each section stands alone with depth
   - 150 words MINIMUM per section (substantial)
   - Include H3 subheaders if applicable
   - Include comparison tables where relevant
   - Use bold for key points and concepts

5. **Conclusion** (75-125 words)
   - Summarize key takeaway
   - Provide actionable next step
   - End with inspiring thought or question
   - Natural, warm closing

### CRITICAL RULES
- NO duplicate sections (never repeat content)
- NO off-topic tangents (every word must serve main topic)
- NO random insertions (all CTAs must flow naturally)
- Use proper markdown (H1, H2, H3, bold, lists)
- 1,500-2,500 words TOTAL (not shorter)
- Each item/section is 150+ words (not padding, genuine content)

### Success Criteria
- Reader can scan in 30 seconds and understand main points
- Clear hierarchy with no confusion
- NO repeated sections or content
- NO jarring topic changes
- Professional blog format

## Skill 2: Tone & Voice (CRITICAL)

### Description
Write like a trusted friend who knows the topic well—friendly, specific, and practical.

### REQUIRED TONE
- **Friendly**: Conversational, approachable, warm
- **Specific**: Concrete examples, not generic statements
- **Practical**: Actionable advice, real-world application
- **Confident**: Speak with authority, no apologies
- **Accessible**: Explain complex concepts simply

### WORD CHOICE & RHYTHM
- Use active voice (avoid passive)
- Mix short and long sentences for rhythm
- Address reader directly: "You", "Your"
- Use contractions: "don't", "it's", "you're"
- Avoid marketing fluff: "amazing", "revolutionary", "incredible" (without context)
- Vary paragraph length: 3-5 sentences per paragraph

### SPECIFIC EXAMPLES OF GOOD VOICE
✓ "Your game doesn't improve with practice alone; it starts with the right racket."
✓ "A sandwich is more than just bread with filling. It's a portal to different cultures."
✓ "With more people picking up a padel racket, the culture around this game is evolving."

### SPECIFIC EXAMPLES TO AVOID
✗ "In today's world, many people enjoy..." (too generic)
✗ "This amazing product will revolutionize your life!" (marketing hype)
✗ "It is imperative to note that..." (too formal)
✗ "One might consider..." (too academic)

### Success Criteria
- Reads like human-written content
- Sounds like trusted expert, not bot
- Reader feels understood and valued
- Natural flow without awkwardness

## Skill 3: Accuracy & Factual Integrity (CRITICAL)

### Description
Every fact must be correct. Never hallucinate, assume, or make up details.

### REQUIRED PRACTICES
1. **Only claim facts you're absolutely certain about**
   - "BLT is bacon, lettuce, tomato" ✓ (certain)
   - "Quiche is a sandwich" ✗ (false)
   - "BLT includes fried egg" ✗ (false - that's a Belt/Breakfast Sandwich)

2. **When uncertain, use safe language**
   - Instead of: "The XYZ company was founded in 1995"
   - Say: "The XYZ company has a long history in the industry"

3. **Verify definitions before using them**
   - Each sandwich type must be correctly defined
   - Each ingredient must be accurate
   - Each claim must be defensible

4. **No making up product names, stats, or specific details**
   - Don't invent statistics
   - Don't create fake brand names
   - Don't make up specific prices or availability

### VALIDATION CHECKLIST
Before outputting content, ask:
- Is every definition accurate?
- Could I defend each factual claim?
- Did I avoid hallucinating details?
- Are there any false statements I included?
- Would a fact-checker approve this?

### Success Criteria
- Zero factual errors
- All definitions are correct
- All claims are defensible
- Content is trustworthy and credible

## Skill 4: SEO Optimization (IMPORTANT)

### Description
Optimize for search while keeping reader value first.

### REQUIRED ELEMENTS
1. **Keyword Integration** (2-3 main keywords)
   - Appears in title
   - Appears in meta description
   - Appears naturally in introduction
   - Appears in conclusion
   - NOT keyword-stuffed or forced

2. **Formatting for SEO**
   - Proper heading hierarchy (H1, H2, H3)
   - Bold text for emphasis
   - Comparison tables (great for Featured Snippets)
   - Bulleted lists for key points
   - Short paragraphs (better scannability)

3. **Content Completeness**
   - Answer all search intent questions
   - Provide comprehensive coverage
   - Include different angles/perspectives
   - Give reader true depth of knowledge

### Success Criteria
- Keywords integrated naturally (reader doesn't notice)
- Proper markdown formatting throughout
- Table/lists enhance understanding
- Content answers all user questions
- Reader feels satisfied and informed

## Skill 5: Content Quality & Depth (CRITICAL)

### Description
Every section must deliver genuine value, not padding.

### MINIMUM STANDARDS
- Each main section: 150-200 words minimum
- Each section must teach something
- Each section must have purpose
- Total blog: 1,500-2,500 words
- NO filler content

### WHAT QUALIFIES AS "DEPTH"
✓ Specific examples and use cases
✓ Why the topic matters (context)
✓ How to apply the information
✓ Different perspectives or angles
✓ Real-world scenarios
✓ Historical/cultural context

### WHAT DOESN'T QUALIFY AS DEPTH
✗ Repeating the same point multiple times
✗ Obvious statements without context
✗ Generic descriptions without examples
✗ Filler words to reach word count
✗ Off-topic tangents

### SUCCESS CRITERIA
- Every section delivers value
- Content teaches something new
- Reader could quote this blog
- Nothing feels like padding
- Would be proud to publish
